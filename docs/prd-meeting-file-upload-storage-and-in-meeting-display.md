# PRD: Загрузка файла встречи для хранения, обработки и отображения внутри встречи

**Дата**: 2026-07-09
**Статус**: Draft

## Цель

Дать участникам встречи возможность загружать файлы (запись встречи и произвольные вложения/материалы), надёжно хранить их и просматривать/воспроизводить прямо внутри интерфейса встречи, без скачивания на устройство.

## Пользовательские сценарии

- Участник встречи выбирает файл записи встречи (видео/аудио) и загружает его -> файл сохраняется и появляется в списке файлов встречи.
- Участник встречи загружает произвольный файл-вложение (документ, презентация, изображение и т.п.) -> файл сохраняется и появляется в списке файлов встречи.
- Участник открывает встречу -> видит список всех загруженных файлов этой встречи с их статусом обработки (в очереди / обрабатывается / готово / ошибка).
- Участник нажимает на загруженный видео/аудио файл -> файл воспроизводится встроенным плеером внутри интерфейса встречи.
- Участник нажимает на загруженный файл другого типа (документ, изображение) -> файл открывается во встроенном превью внутри интерфейса встречи.
- Участник загружает аудио/видео запись встречи -> после завершения обработки в интерфейсе встречи появляется текстовая расшифровка (транскрипция), связанная с этим файлом.
- Загрузка файла завершается ошибкой (сеть, превышен размер, неподдерживаемый формат) -> участник видит сообщение об ошибке и может повторить загрузку.

## В скоупе

- Загрузка файла на встречу любым участником этой встречи (любой, кто состоит в списке участников).
- Два типа контента: запись встречи (видео/аудио) и произвольные вложения/материалы встречи (документы, изображения и т.п.).
- Надёжное хранение загруженных файлов, связанных с конкретной встречей.
- Список загруженных файлов встречи, видимый всем участникам этой встречи, с статусом обработки каждого файла.
- Транскрипция/расшифровка для аудио- и видеозаписей встречи.
- Конвертация/транскодирование загруженных аудио/видео файлов к единому формату хранения и воспроизведения.
- Отображение файлов внутри интерфейса встречи через встроенный плеер (аудио/видео) и превью (прочие поддерживаемые типы), без обязательного скачивания.
- Обработка ошибок загрузки (сетевые сбои, превышение лимита размера, неподдерживаемый формат) с понятной обратной связью пользователю.
- Ограничения на допустимые типы и максимальный размер файла.

## Не в скоупе

- Скачивание/экспорт файлов за пределы интерфейса встречи (кнопка "Скачать") — только просмотр/воспроизведение внутри встречи.
- Редактирование содержимого файлов (обрезка видео, аннотирование документов и т.п.).
- Совместное редактирование или комментирование файлов в реальном времени.
- Автоматическая запись встречи "на лету" самим сервисом (эта фича — только про загрузку уже готового файла записи, не про захват экрана/звука во время звонка).
- Поиск по содержимому транскрипции или файлов.
- Права доступа тоньше, чем "участник встречи" (например, роли "только просмотр"/"только загрузка").
- Удаление/управление версиями загруженных файлов.
- Уведомления (push/email) о завершении обработки файла.

## Технические ограничения

- Обработка файлов (транскодирование, транскрипция) не может быть синхронной частью HTTP-запроса загрузки — требуется асинхронный статус обработки, который видит пользователь.
- Список участников и права доступа к встрече уже определены существующим модулем `meetings` (host + participants) — доступ к файлам встречи должен опираться на эту существующую модель, не вводить отдельную.

## Критерии готовности

- [ ] Любой участник встречи может загрузить файл записи встречи (видео/аудио) и файл-вложение произвольного типа.
- [ ] Загруженный файл появляется в списке файлов встречи с корректным статусом обработки для всех участников встречи.
- [ ] Видео/аудио файлы воспроизводятся встроенным плеером внутри интерфейса встречи без скачивания.
- [ ] Файлы других поддерживаемых типов открываются встроенным превью внутри интерфейса встречи без скачивания.
- [ ] После завершения обработки аудио/видеозаписи в интерфейсе встречи доступна текстовая расшифровка, связанная с этим файлом.
- [ ] Загруженный аудио/видео файл проходит конвертацию/транскодирование в единый формат хранения перед тем, как становится доступен для воспроизведения.
- [ ] При ошибке загрузки (сеть, размер, формат) пользователь видит понятное сообщение об ошибке и может повторить попытку.
- [ ] Попытка загрузить файл недопустимого типа или превышающий лимит размера отклоняется с явным объяснением причины.
